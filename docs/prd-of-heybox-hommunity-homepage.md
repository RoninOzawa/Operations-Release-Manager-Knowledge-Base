# 2 背景与目标
**背景**
随着小黑盒完成早期工具型用户积累，平台需要提升用户日常打开频次与社区内容消费深度。仅依赖游戏库、战绩查询、折扣购买等工具场景，难以形成稳定的日活和关系链沉淀。（补充判断数据：小黑盒APP的平均用户市场停滞在xx分钟左右、CDK的购买率仅为xx%等）
因此，小黑盒通过首页信息流、社区分区、帖子互动、关注推荐和生活化内容扩展，将玩家从“查工具”引导到“逛社区”，提高用户留存，进一步引导黑盒商城的内容消费。

**目标**
本次社区版本更新，旨在将小黑盒从低频工具型产品进一步升级为高频玩家社区。通过内容消费、社区互动、玩家身份展示、圈层聚合和消费转化等能力，提升用户日常打开频次与社区留存，形成围绕游戏玩家身份展开的垂直社区生态。
1. **提升打开频次**：从查战绩、看折扣、同步游戏库等低频工具场景，扩展到刷帖子、看资讯、参与讨论、关注话题等高频社区场景，提升用户访问频次与使用时长。
2. **沉淀玩家身份**：通过绑定 Steam 游戏库，展示用户游戏数量、游戏时长、发帖记录、互动数据、黑盒等级等信息，让用户在平台内形成可识别、可积累的玩家身份。
3. **强化圈层归属**：通过游戏圈子、话题划分、关注关系和内容推荐，让同游戏、同兴趣、同消费偏好的玩家聚集在一起，形成稳定的讨论氛围和社区关系，增强用户的平台依赖。
4. **承接商业转化**：通过内容种草、评价讨论、折扣提醒、商城活动和福利任务，将社区流量转化为游戏购买、活动参与、广告曝光和合作内容消费等商业价值。

## 3.1 用户与运营场景
**用户与运营场景**
- C端浅度用户（张三）：张三在地铁、用餐等碎片化时间打开小黑盒社区，通过首页热门帖子、关注的内容创作者、平台折扣活动快速获取感兴趣的内容，并可能进行点赞、收藏、评论等轻量互动。该场景主要满足用户“随手逛、找内容、看讨论”的高频社区消费需求，逐渐培养用户打开习惯，并推动其向深度用户转化。
- C 端深度用户（王五）：王五已绑定 Steam 游戏库，并长期在小黑盒内浏览游戏社区、参与话题讨论、发布帖子和评论。他的游戏数量、游戏时长、黑盒等级、发帖记录等信息沉淀在个人主页中，形成可展示、可积累的玩家身份。随着平台资产沉淀加深，用户的平台黏性与迁移成本提升；同时，王五可通过互动行为获得 H 币、经验值或等级成长，并在商城、福利活动等场景中使用平台资产，完成从社区活跃到商业转化的链路。
- B端运营（李四）：李四负责小黑盒社区内容与秩序运营。他通过后台查看社区内容数据、举报记录和高风险评论，对违规内容进行审核、静默处理、删除、封禁等操作；同时根据社区热点、游戏热度和活动节奏，配置推荐内容、话题分区、圈子入口和运营活动，保障社区内容质量、讨论氛围和用户活跃度。

7 补充文档：小黑盒产品功能的演进

2017年以前，Steam的中国玩家群体主要集中在《Dota 2》和《CS:GO》两款网游上。小黑盒的前身 DOTA MAX 是一款 Dota 2 数据查询工具，通过贴吧等社区渠道积累了早期用户。
2017年3月，《绝地求生》登陆Steam，迅速在中国市场引发热潮。Steam平台10月的硬件调查报告显示，简体中文用户占比首次突破一半，达到56.37%，正式超越英语成为Steam第一大用户语言。
同年7月，小黑盒APP1.0上线，它的核心功能（Steam游戏库同步、战绩查询、游戏CDK商城等）几乎全部是围绕Steam生态和"吃鸡"玩家的需求来设计的。小黑盒诞生的时间点，恰好卡在这一波用户爆发的前夜。
当时中国游戏社区市场存在以下几个结构性空白：
- **泛社区承接不足**：steam玩家数量激增，贴吧、TGA 等论坛虽然已经形成规模的玩家社群，但缺少稳定、垂直、结构化的深度社区体验。
- **传统游戏媒体社区能力不足**：小黑盒问世前，早有许多老牌游戏社区软件。4399游戏盒、taptap瞄准单一手机游戏市场；游民星空、3DM 等产品在单机资讯和攻略上有优势，但社区关系链、互动体验等不够完善。
- **垂直综合型平台缺位**：市场缺少一个同时具备游戏工具、内容消费、身份展示、社区互动和生活化讨论能力的玩家平台。
- **玩家社区需求泛化**：游戏玩家的需求已经从单一的游戏资讯、攻略查询，扩展到围绕游戏身份的生活化表达与圈层互动。这也是小黑盒后期在内容形态上向小红书等平台靠近的底层逻辑。
