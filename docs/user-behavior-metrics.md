# 1 指标概述
---
我将用户行为指标按照生命周期，分为行为链路和质量评估两个模块：
1) 用户行为漏斗链路：曝光层 → 点击层 → 到达层 → 使用层
2) 质量评估：转化层、活跃层、留存层、付费层。
```txt
质量评估指标不是独立于用户行为链路存在，而是对链路中各行为节点进行关系计算、深度判断和长期价值判断。
```
# 2 用户行为链路
---
## 2.1 曝光层
| 简称        | 全称             | 中文      | 含义                 |
| --------- | -------------- | ------- | ------------------ |
| PV        | Page View      | 页面浏览量   | 页面访问次数             |
| UV        | Unique Visitor | 去重访问用户数 | 页面去重访问人数           |
| IMP       | Impression     | 曝光量     | 资源位/卡片/入口/模块被展示的次数 |
| Reach     | Reach          | 触达人数    | 资源位/卡片/入口/模块访问人数   |
| Frequency | Frequency      | 人均触达频次  | 曝光量 / 触达人数         |
- 曝光层反应网页或资源位的配置效果。
- Q：触达和到达的区别？
```txt
触达：平台把内容送到用户面前。触达通常是被系统展示的内容。
到达：用户主动进入了某个目标场景。到达一般发生在点击之后。

在某些功能上，触达和到达功能会发生重合。
例如，如果这个目标场景本身就是一个页面或模块，那么：用户到达这个页面 = 也被这个页面触达
```
## 2.2 点击层
| 简称          | 全称                 | 中文    | 含义          |
| ----------- | ------------------ | ----- | ----------- |
| Click / CLK | Click              | 点击次数  | 资源位被点击总次数   |
| Click UV    | Click Unique User  | 点击用户数 | 去重点击人数      |
| CTR         | Click Through Rate | 点击率   | 点击次数 / 曝光次数 |
- 点击层反应应用内容对用户的吸引程度。
## 2.3 到达层（承接层）
| 简称  | 全称                | 中文       | 含义                        | 说明                                            |
| --- | ----------------- | -------- | ------------------------- | --------------------------------------------- |
| LPV | Landing PV        | 落地页访问量   | 落地页被访问次数                  | 用户点击后实际进入承接页的次数                               |
| LUV | Landing UV        | 落地页访问用户数 | 去重访问用户数                   | 有多少用户真正到达承接页                                  |
| LPR | Landing Rate      | 到达率      | 落地页访问用户数 / 点击用户数          | 判断点击后是否成功到达<br>如果这个值低，说明技术承接出了问题（比如跳转失败、加载崩溃） |
| BR  | Bounce Rate       | 跳失率      | 只访问落地页但未继续行为的用户 / 落地页访问用户 | 判断页面有没有接住用户<br>如果这个值高，说明内容不匹配或体验太差            |
| LSR | Load Success Rate | 页面加载成功率  | 加载成功次数 / 页面请求次数           | 判断是否有技术承接问题                                   |
| PLT | Load Time         | 页面加载耗时   | 页面从请求到可交互的时间              | 加载慢会影响转化                                      |
| /   | Error Rate        | 错误率      | 页面失败/异常次数 / 请求次数          | 判断承接链路稳定性                                     |
- 到达层反映用户是否成功到达了下一级页面。
## 2.4 使用层
| 简称  | 全称                     | 中文     | 含义                |
| --- | ---------------------- | ------ | ----------------- |
| /   | Session/Session Length | 单次会话时长 | 衡量用户一次连续使用的时长和深度  |
| TS  | Time Spent             | 总使用时长  | 衡量用户对内容累计的消耗量     |
| /   | Average Duration       | 平均时长   | 平均每次或每人使用时长，需注明口径 |
| /   | Starts / Launches      | 启动次数   | 游戏或功能成功启动次数       |
- 使用层反映用户进入目标功能后，实际的使用情况
- 使用层可以分成三类：
	1. 启动意愿：启动点击次数、启动点击用户数
	2. 启动结果：启动成功次数、启动成功用户数、启动成功率、启动失败率
	3. 使用质量：有效游玩用户数、有效游玩率、总游玩时长、人均游玩时长、单次游玩时长

# 3 质量评估
---
## 3.1 转化层
| 简称       | 全称                       | 中文     | 公式           |
| -------- | ------------------------ | ------ | ------------ |
| CVR      | Conversion Rate          | 转化率    | 转化人数 / 前一步人数 |
| CTCVR    | Click To Conversion Rate | 点击到转化率 | 转化人数 / 点击人数  |
| View CVR | View To Conversion Rate  | 曝光到转化率 | 转化人数 / 曝光人数  |
- 转化率是用于衡量用户从一个行为节点进入下一个行为节点的关系指标。它依赖具体业务目标定义，不应脱离行为链路单独使用。因此【转化】最好不要泛泛叫 CVR，要写清楚目标：
	曝光到启动转化率
	点击到启动转化率
	点击到有效游玩转化率
	……
## 3.2 活跃层
| 简称       | 全称                          | 中文       |
| -------- | --------------------------- | -------- |
| DAU      | Daily Active Users          | 日活跃用户数   |
| WAU      | Weekly Active Users         | 周活跃用户数   |
| MAU      | Monthly Active Users        | 月活跃用户数   |
| DNU / NU | Daily New Users / New Users | 日新增用户    |
| ACU      | Average Concurrent Users    | 平均同时在线人数 |
| PCU      | Peak Concurrent Users       | 最高同时在线人数 |
- 活跃层反映平台的用户规模。
## 3.3 留存层
| 简称             | 全称               | 中文    |
| -------------- | ---------------- | ----- |
| Retention / RR | Retention Rate   | 留存率   |
| D1 Retention   | Day 1 Retention  | 次日留存  |
| D7 Retention   | Day 7 Retention  | 7日留存  |
| D30 Retention  | Day 30 Retention | 30日留存 |
| Churn          | Churn Rate       | 流失率   |
- 留存层反映平台功能是否符合匹配用户需求。
## 3.4 付费层
|简称|全称|中文|
|---|---|---|
|PU|Paying Users|付费用户数|
|Pay Rate|Paying Rate|付费率|
|ARPU|Average Revenue Per User|每活跃用户平均收入|
|ARPPU|Average Revenue Per Paying User|每付费用户平均收入|
|ARPDAU|Average Revenue Per Daily Active User|日活用户平均收入|
|LTV|Lifetime Value|用户生命周期价值|
|ROI|Return on Investment|投入产出比|
|ROAS|Return on Ad Spend|广告花费回报|
- 付费层反映用户的商业价值。
