# 1 数据漏斗概述
---
- 数据漏斗就是把用户完成目标前的关键步骤串起来，逐层观察转化和流失，用来判断问题到底发生在哪一步。
- 数据漏斗的核心在于，**告诉运营用户在哪一步流失了**。
- 漏斗里的每个关键步骤，都需要有可观测的数据事件支撑，因此通常都应该对应一个埋点事件。这个事件可以是单独埋点，也可以由其他埋点组合计算。因此一个好的埋点设计必须符合两个原则：
	1）关键行为必须埋
	2）可由已有事件稳定计算的指标，不重复埋
## 1.1 可复用漏斗模型
- 一个可复用的资源位漏斗大概如下
```txt
资源位曝光
→ 资源位点击
→ 详情页访问
→ 点击开始游戏
→ 游戏启动成功
→ 有效游玩
→ 次日复玩

对应的数据埋点就是：
曝光量 resource_exposure
→ 点击量 resource_click
→ 详情页访问量 game_detail_view
→ 启动点击量 game_start_click
→ 启动成功量 game_start_success
→ 有效游玩量 game_play_effective（可换算，由 game_end.play_duration 判断）
→ 次日复玩量 game_replay_next_day（可换算，由次日 game_start_success 判断）

对应指标是：
点击率 = 点击量 / 曝光量
详情页到达率 = 详情页访问量 / 点击量
启动转化率 = 启动成功量 / 点击量
有效游玩率 = 有效游玩量 / 启动成功量
复玩率 = 次日复玩用户 / 当日启动用户
```
## 1.2 数据漏斗分析示例
```txt
资源位曝光
10000 人
↓
资源位点击
800 人
↓
进入详情页
700 人
↓
点击开始游戏
400 人
↓
启动成功
350 人
↓
有效游玩
250 人
↓
次日复玩
60 人

这就是一个资源位效果漏斗。拆解下来就是：
看到资源位的人很多
→ 点击的人少一些
→ 启动游戏的人更少
→ 真正玩超过几分钟的人再少一些
→ 第二天还回来玩的人更少

我们该怎么分析这个数据呢？
比如：
曝光 10000，点击 200
说明资源位吸引力弱，可能是：
	游戏不够热门
	素材不好
	位置不佳
	标题不吸引人
	用户不感兴趣

再比如：
点击 800，启动 100
说明用户点了但没有真正开始玩，问题可能在：
	详情页承接弱
	启动路径太长
	需要下载/排队
	游戏不符合预期
	按钮不明显

再比如：
启动 350，有效游玩 80
说明用户进去了但很快走，问题可能是：
	游戏质量
	适配问题
	加载慢
	首局体验差
	操作不顺
```
